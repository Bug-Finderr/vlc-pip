# VLC PiP (reshape-real-window) Implementation Plan - v1, C#

> Executed 2026-07-02; shipped as tag `v1.0.0`, then replaced by the Rust rewrite ([2026-07-02-rust-rewrite.md](2026-07-02-rust-rewrite.md)). The C# implementation is archived at the tag; this plan's full code listings are preserved in git history (commit `48aab44`). What remains here is the decisions.

**Goal:** Toggle the real VLC 3.0.23 window into a borderless, always-on-top, corner-parked PiP from VLC's own View menu (and Ctrl+Alt+P), restoring the exact original geometry/styles on exit.

**Architecture:** A VLC Lua extension (`capabilities={"trigger"}`) writes `toggle` to a request file; a long-lived GUI-subsystem C# daemon (started at login) polls that file every ~150 ms and also owns a global hotkey and low-level keyboard/mouse hooks (fullscreen prevention). All Win32 reshaping (`SetWindowLongPtr` + `SetWindowPos(SWP_FRAMECHANGED)`, `HWND_TOPMOST`) happens in the helper. Single source of truth for "in PiP" = presence of `%TEMP%\vlc-pip.json`.

**Tech stack:** C# on .NET 10 (LTS, user-scoped install), pure Win32 P/Invoke with NO WinForms - a plain `GetMessage` pump with `RegisterHotKey(NULL)` + `SetTimer(NULL)` replaces the originally-specced ApplicationContext workaround (`WM_HOTKEY` is posted to the thread queue per MS docs); xUnit for pure-logic tests, VLC Lua for the trigger, PowerShell for install/uninstall/smoke scripts.

**Why C# and not TypeScript (user default):** the deliverable is low-level Win32 (LL hooks, global hotkey, window style surgery) in a single GUI-subsystem exe with a message pump. Node needs fragile native FFI addons and cannot host LL hooks cleanly; C# P/Invoke is the established fit.

## Decisions that outlived v1

Most constraints this plan pinned were promoted into SPEC §3/§7/§8 (trigger capability, lazy `os.getenv`, only `.lua` in the extensions folder, `SWP_FRAMECHANGED` after style changes, prevent-don't-snap-back fullscreen guards, request file + login-started daemon for flash-free toggling). C#-specific and gone with v1: hook delegates must live in static fields or the GC collects them mid-hook.

Declared omission: the optional minimal look (Ctrl+H-like, `SetWindowRgn`) was YAGNI'd out of initial v1 scope; it landed later in v1's audit rounds and is contract in v2 (SPEC §7 maintain_region).
