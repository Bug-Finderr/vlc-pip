# v2.1 Gestures - Design & Plan

Drag-to-move, edge/corner drag-resize, and size/corner persistence for the v2 Rust daemon.
Decided with the owner 2026-07-02: free placement on release (no corner snap), double-click stays inert, no wheel interception (Ctrl+wheel resize designed then dropped - drag-resize was the intent), opacity rejected (unverified D3D11 rendering in layered windows). Drags starting in the outer 16px band of the visible PiP resize (aspect-locked, live, Firefox-style); center drags move.

Plan verified 2026-07-02 by a 3-agent pass (compile spike ran Tasks 1-3 in a scratch copy: 41/41 tests green; integration + coverage agents each read the real sources); all findings folded in below.

## Constraints (researched 2026-07-02, sources fetched)

- Single left click on video is a no-op in stock VLC 3.x ([events.c](https://raw.githubusercontent.com/videolan/vlc/3.0.x/modules/video_output/win32/events.c)) - drag needs no modifier and no click swallowing.
- Plain wheel = VLC volume, Ctrl+wheel = subtitle scale ([libvlc-module.c](https://raw.githubusercontent.com/videolan/vlc/3.0.x/src/libvlc-module.c)); Windows "scroll inactive windows" ships on, so wheel-volume over the PiP already works - document it, never touch the wheel.
- A swallowed LL button-down never updates async button state, so a native drag loop can't follow a swallowed click; we drive `SetWindowPos(SWP_ASYNCWINDOWPOS)` ourselves per mouse-move (the [AltSnap](https://github.com/RamonUnch/AltSnap) model, [SetWindowPos](https://learn.microsoft.com/en-us/windows/win32/api/winuser/nf-winuser-setwindowpos)).
- Native edge-resize is out: aspect lock needs [WM_SIZING](https://learn.microsoft.com/en-us/windows/win32/winmsg/wm-sizing) in VLC's own WndProc, and the minimal-look region clips the thick-frame hit zones anyway.
- `plan_region` only repositions during a size correction (existing code), so free placement survives convergence with zero changes to it.

## Design

`Corner` becomes "nearest corner as of the last gesture": derived on drag release, consumed only by the next enter and any convergence-driven re-park. `plan_region` and its six tests stay byte-identical.

New pure fns in `geometry.rs`, unit-tested: `nearest_corner(win, work)` (center quadrant, tie = `br`); `classify_zone(pt, visible, band) -> DragZone` (`Interior` | 4 edges | 4 corners; corners win; band = 16px DPI-scaled); `plan_resize(start, zone, dx, dy, work) -> Rect` (opposite corner/edge anchored, aspect-locked, clamped to 256px wide .. 80% of work area).

**Hook** (`mouse_hook`; nothing new swallowed, injected input not filtered): an allowed button-down over the PiP arms with cursor origin, window rect, and zone (visible rect = window rect intersected with the region box). Mouse-move past `SM_CX/CYDRAG` activates move (interior) or resize (band); while active, store `LATEST_*` and post one coalesced `WM_APP_DRAG`. Button-up posts `WM_APP_DRAGEND` and resets. Burst-swallowed downs never arm (fullscreen guard wins). Idle mouse-move cost = one atomic load. A generation counter rides the message lParam so a rapid release-and-repress can't mix a stale message with re-armed statics.

**Pump**: `WM_APP_DRAG` - guard `CACHED_HWND == DRAG_HWND != 0` and generation; move = `SetWindowPos(NOSIZE|NOZORDER|NOACTIVATE|ASYNCWINDOWPOS|NOSENDCHANGING)` at start + delta; resize = drop the region once, then apply `plan_resize` (same flags minus `NOSIZE`). `WM_APP_DRAGEND` - guard `owns_state`; finalize from the pump's own computed rect (the async apply hasn't landed in VLC yet - never re-measure): move: derive corner, save state + config, reset tracker; resize: first set `TargetW/H` = final size minus chrome-at-drag-start (`min=1`; no region = chrome 0, which also covers `min=0`), then same. `maintain_region` is skipped while a drag is active. Convergence re-clips after a resize with at most a +/-2px correction.

**Persistence**: `%APPDATA%\vlc\pip\config.txt`, one line of existing option tokens (`w=640 h=360 c=br`), written on drag-end (pump, never the hook, failures swallowed), read at every enter as defaults < config < argv via `parse_options` (later-wins is already pinned by test). Missing/unreadable config = exact v2.0 behavior.

```rust
/// Plain rect so this module stays windows-sys-free (native.rs converts at the boundary).
#[derive(Debug, Clone, Copy, PartialEq)]
pub struct Rect {
    pub left: i32,
    pub top: i32,
    pub right: i32,
    pub bottom: i32,
}

/// Where within the visible PiP a drag started; stored in an AtomicU8 by the hook.
#[derive(Debug, Clone, Copy, PartialEq)]
#[repr(u8)]
pub enum DragZone {
    Interior = 0,
    Left = 1,
    Right = 2,
    Top = 3,
    Bottom = 4,
    TopLeft = 5,
    TopRight = 6,
    BottomLeft = 7,
    BottomRight = 8,
}

impl DragZone {
    pub fn from_u8(v: u8) -> DragZone {
        use DragZone::*;
        match v {
            1 => Left,
            2 => Right,
            3 => Top,
            4 => Bottom,
            5 => TopLeft,
            6 => TopRight,
            7 => BottomLeft,
            8 => BottomRight,
            _ => Interior,
        }
    }
}

/// Work-area quadrant of the window center. Ties resolve toward "br" (codebase fallback).
pub fn nearest_corner(win: &Rect, work: &Rect) -> &'static str {
    let cx = win.left + (win.right - win.left) / 2;
    let cy = win.top + (win.bottom - win.top) / 2;
    let left_half = cx < work.left + (work.right - work.left) / 2;
    let top_half = cy < work.top + (work.bottom - work.top) / 2;
    match (left_half, top_half) {
        (true, true) => "tl",
        (false, true) => "tr",
        (true, false) => "bl",
        (false, false) => "br",
    }
}

/// Zone of a point in the visible rect: outer `band` px = resize, else move. Corners win.
pub fn classify_zone(x: i32, y: i32, vis: &Rect, band: i32) -> DragZone {
    let l = x < vis.left + band;
    let r = x >= vis.right - band;
    let t = y < vis.top + band;
    let b = y >= vis.bottom - band;
    match (l, r, t, b) {
        (true, _, true, _) => DragZone::TopLeft,
        (_, true, true, _) => DragZone::TopRight,
        (true, _, _, true) => DragZone::BottomLeft,
        (_, true, _, true) => DragZone::BottomRight,
        (true, ..) => DragZone::Left,
        (_, true, ..) => DragZone::Right,
        (_, _, true, _) => DragZone::Top,
        (_, _, _, true) => DragZone::Bottom,
        _ => DragZone::Interior,
    }
}
```

```rust
/// New window rect for a live resize drag. The dominant relative delta drives the scale
/// (edges have one axis by construction); the other dimension follows start's aspect,
/// including at the clamps. i64 products: screen coords can make i32 overflow.
pub fn plan_resize(start: &Rect, zone: DragZone, dx: i32, dy: i32, work: &Rect) -> Rect {
    use DragZone::*;
    let (w0, h0) = (start.right - start.left, start.bottom - start.top);
    if w0 < 1 || h0 < 1 {
        return *start; // garbage measurement: no-op
    }
    let dw = match zone {
        Right | TopRight | BottomRight => dx,
        Left | TopLeft | BottomLeft => -dx,
        _ => 0,
    };
    let dh = match zone {
        Bottom | BottomLeft | BottomRight => dy,
        Top | TopLeft | TopRight => -dy,
        _ => 0,
    };
    let width_driven = i64::from(dw.abs()) * i64::from(h0) >= i64::from(dh.abs()) * i64::from(w0);
    let min_w = 256;
    let max_w = ((work.right - work.left) * 4 / 5)
        .min((i64::from(work.bottom - work.top) * 4 / 5 * i64::from(w0) / i64::from(h0)) as i32)
        .max(min_w); // tiny work area: clamp() must never see min > max
    let raw_w = if width_driven { w0 + dw } else { (i64::from(h0 + dh) * i64::from(w0) / i64::from(h0)) as i32 };
    let w = raw_w.clamp(min_w, max_w);
    let h = (i64::from(w) * i64::from(h0) / i64::from(w0)) as i32;
    let (left, right) = match zone {
        Left | TopLeft | BottomLeft => (start.right - w, start.right),
        Right | TopRight | BottomRight => (start.left, start.left + w),
        _ => {
            let l = start.left + (w0 - w) / 2;
            (l, l + w)
        }
    };
    let (top, bottom) = match zone {
        Top | TopLeft | TopRight => (start.bottom - h, start.bottom),
        Bottom | BottomLeft | BottomRight => (start.top, start.top + h),
        _ => {
            let t = start.top + (h0 - h) / 2;
            (t, t + h)
        }
    };
    Rect { left, top, right, bottom }
}
```

```rust
pub fn window_rect(h: isize) -> RECT {
    unsafe {
        let mut r: RECT = std::mem::zeroed();
        GetWindowRect(hw(h), &mut r);
        r
    }
}

// The minimal-look region clips painting AND hit-testing, so the gesture surface is the
// region box, not the window rect.
pub fn visible_rect(h: isize) -> RECT {
    unsafe {
        let wr = window_rect(h);
        let probe = CreateRectRgn(0, 0, 0, 0);
        let mut b: RECT = std::mem::zeroed();
        let vis = if GetWindowRgn(hw(h), probe) != 0 && GetRgnBox(probe, &mut b) > NULLREGION {
            RECT { left: wr.left + b.left, top: wr.top + b.top, right: wr.left + b.right, bottom: wr.top + b.bottom }
        } else {
            wr
        };
        DeleteObject(probe);
        vis
    }
}

pub fn drag_band(h: isize) -> i32 {
    let dpi = unsafe { GetDpiForWindow(hw(h)) };
    if dpi == 0 { 16 } else { 16 * dpi as i32 / 96 }
}

pub fn drag_move(h: isize, x: i32, y: i32) {
    unsafe {
        SetWindowPos(hw(h), std::ptr::null_mut(), x, y, 0, 0,
            SWP_NOSIZE | SWP_NOZORDER | SWP_NOACTIVATE | SWP_ASYNCWINDOWPOS | SWP_NOSENDCHANGING);
    }
}

pub fn drag_resize(h: isize, r: &geometry::Rect) {
    unsafe {
        if has_region(h) {
            SetWindowRgn(hw(h), std::ptr::null_mut(), 1); // mini chrome shows; release re-clips
        }
        SetWindowPos(hw(h), std::ptr::null_mut(), r.left, r.top, r.right - r.left, r.bottom - r.top,
            SWP_NOZORDER | SWP_NOACTIVATE | SWP_ASYNCWINDOWPOS | SWP_NOSENDCHANGING);
    }
}

/// Adopt the drag result: Corner = nearest as of `fin`; a resize also adopts the new video
/// size (`fin` minus chrome measured at drag start). State first, then config.
pub fn finish_drag(fin: &geometry::Rect, resized: bool, chrome_w: i32, chrome_h: i32) -> bool {
    let path = state::state_path();
    let Some(mut s) = state::load(&path) else { return false };
    if !owns_state(&s) {
        return false; // VLC died mid-drag: next tick's maintain_region cleans up
    }
    let wa = work_area(s.hwnd as isize);
    let work = geometry::Rect { left: wa.left, top: wa.top, right: wa.right, bottom: wa.bottom };
    s.corner = geometry::nearest_corner(fin, &work).to_string();
    if resized {
        let (tw, th) = (fin.right - fin.left - chrome_w, fin.bottom - fin.top - chrome_h);
        if tw > 0 && th > 0 {
            s.target_w = tw;
            s.target_h = th;
        }
    }
    let ok = state::save(&s, &path).is_ok();
    crate::options::save_config(s.target_w, s.target_h, &s.corner);
    ok
}
```
