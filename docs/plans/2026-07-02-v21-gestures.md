# v2.1 Gestures - Design & Plan

Drag-to-move, edge/corner drag-resize, and size/corner persistence for the v2 Rust daemon.
Decided with the owner 2026-07-02: free placement on release (no corner snap), double-click stays inert, no wheel interception (Ctrl+wheel resize designed then dropped - drag-resize was the intent), opacity rejected (unverified D3D11 rendering in layered windows). Drags starting in the outer 16px band of the visible PiP resize (aspect-locked, live, Firefox-style); center drags move.

## Constraints (researched 2026-07-02, sources fetched)

- Single left click on video is a no-op in stock VLC 3.x ([events.c](https://raw.githubusercontent.com/videolan/vlc/3.0.x/modules/video_output/win32/events.c)) - drag needs no modifier and no click swallowing.
- Plain wheel = VLC volume, Ctrl+wheel = subtitle scale ([libvlc-module.c](https://raw.githubusercontent.com/videolan/vlc/3.0.x/src/libvlc-module.c)); Windows "scroll inactive windows" ships on, so wheel-volume over the PiP already works - document it, never touch the wheel.
- A swallowed LL button-down never updates async button state, so a native drag loop can't follow a swallowed click; we drive `SetWindowPos(SWP_ASYNCWINDOWPOS)` ourselves per mouse-move (the [AltSnap](https://github.com/RamonUnch/AltSnap) model, [SetWindowPos](https://learn.microsoft.com/en-us/windows/win32/api/winuser/nf-winuser-setwindowpos)).
- Native edge-resize is out: aspect lock needs [WM_SIZING](https://learn.microsoft.com/en-us/windows/win32/winmsg/wm-sizing) in VLC's own WndProc, and the minimal-look region clips the thick-frame hit zones anyway.
- `plan_region` only repositions during a size correction (existing code), so free placement survives convergence with zero changes to it.
- No new Cargo features: every new symbol (`PostThreadMessageW`, `GetRgnBox`, `GetDpiForWindow`, `GetCurrentThreadId`, the `SWP_*`/`SM_*DRAG`/`WM_APP` constants) verified present in windows-sys 0.61.2 under the already-enabled gates.

## Design

`Corner` becomes "nearest corner as of the last gesture": derived on drag release, consumed only by the next enter and any convergence-driven re-park. `plan_region` and its six tests stay byte-identical.

New pure fns in `geometry.rs`, unit-tested: `nearest_corner(win, work)` (center quadrant, tie = `br`); `classify_zone(pt, visible, band) -> DragZone` (`Interior` | 4 edges | 4 corners; corners win; band = 16px DPI-scaled); `plan_resize(start, zone, dx, dy, work) -> Rect` (opposite corner/edge anchored, aspect-locked, clamped to 256px wide .. 80% of work area).

**Hook** (`mouse_hook`; nothing new swallowed, injected input not filtered): an allowed button-down over the PiP arms with cursor origin, window rect, and zone (visible rect = window rect intersected with the region box). Mouse-move past `SM_CX/CYDRAG` activates move (interior) or resize (band); while active, store `LATEST_*` and post one coalesced `WM_APP_DRAG`. Button-up posts `WM_APP_DRAGEND` and resets. Burst-swallowed downs never arm (fullscreen guard wins). Idle mouse-move cost = one atomic load. Both messages carry mode in wParam and a generation counter in lParam, so a rapid release-and-repress can't mix a stale message with re-armed statics.

**Pump**: `WM_APP_DRAG` - guard `CACHED_HWND == DRAG_HWND != 0` and generation; compute the target rect itself (move = start + delta; resize = `plan_resize`, dropping the region once so the live resize shows mini chrome until release re-clips) and apply async. `WM_APP_DRAGEND` - guard `owns_state`, finalize FROM THE COMPUTED RECT (the async `SetWindowPos` hasn't landed in VLC yet - a fresh `GetWindowRect` reads stale): derive corner, and for resizes set `TargetW/H` = final size minus chrome-at-drag-start (chrome = start window minus start visible; no region = 0, which also covers `min=0`); save state + config, reset the region tracker. `maintain_region` is skipped while a drag is active. Convergence re-clips after a resize with at most a +/-2px correction.

**Persistence**: `%APPDATA%\vlc\pip\config.txt`, one line of existing option tokens (`w=640 h=360 c=br`), written on drag-end (pump, never the hook, failures swallowed), read at every enter as defaults < config < argv via `parse_options` (later-wins is already pinned by test). Missing/unreadable config = exact v2.0 behavior.

## Accepted edges

No sizing cursors over the band (cursor feedback needs injection; Firefox PiP is equally unmarked); VLC's mini chrome shows during a resize drag until release re-clips; outer 16px is resize territory; click-then-redrag within double-click time doesn't arm (burst guard wins, self-heals); raw-pixel sizes across mixed-DPI monitors; VLC dying mid-drag dissolves on the guards.

## Plan

Verified 2026-07-02 by a 3-agent pass: a compile spike ran tasks 1-3 verbatim in a scratch copy (41/41 tests, every expectation numerically correct); integration + coverage agents read the real sources. Their findings are folded into the task notes below. The full-code draft of this plan is preserved in git history (`cdba4aa`) if ever needed.

1. **geometry.rs - zones + corner** (5 tests): `Rect` (plain struct, keeps the module windows-sys-free), `DragZone` (`repr(u8)` + `from_u8`), `nearest_corner` (tie = br), `classify_zone` (boundaries: `< left+band`, `>= right-band`).
2. **geometry.rs - plan_resize** (9 tests): dominant relative delta drives (i64 products - screen coords overflow i32), other dimension follows start aspect including at clamps; max width floors to min so `clamp()` never panics on tiny work areas; pure edges keep the perpendicular center fixed.
3. **options.rs - persistence** (3 tests): `config_path`, `merge(cfg, argv)`, `effective(argv)` (re-reads per call - the daemon must see its own gesture writes), `save_config`; main.rs switches to `effective(&args[1..])`.
4. **native.rs - drag primitives**: `work_area` goes pub; `window_rect`, `visible_rect` (region box offset by window origin), `drag_band` (16 x dpi/96, fallback 16), `drag_move`/`drag_resize` (async flags + NOSENDCHANGING; resize drops the region first), `finish_drag(fin, resized, chrome_w, chrome_h)` taking the caller's computed rect.
5. **daemon.rs - hook side**: statics + `DRAG_GEN` bumped at arm; arm goes right after the `LAST_ALLOWED_*` stores; new `WM_MOUSEMOVE` arm (idle = one load); button-up: REPLACE the whole `WM_LBUTTONUP && SWALLOW_NEXT_UP` arm - the swallow check is fused into that condition today and must move into the body AFTER drag-end handling, else either drags never end or double-click up-pairing breaks (verification blocker).
6. **daemon.rs - pump side**: `run(argv)` + `poll_request(argv)` with `effective` inside the toggle/enter arms only; timer arm skips `maintain_region` during drags (fresh tracker); `WM_APP` arm per the Design pump paragraph. From here clippy must be FULLY clean - tasks 1-3 leave expected `dead_code` warnings until their consumers land (bin crate; verified empirically).
7. **smoke-test.ps1** (10 new checks, 31 total): park config AFTER the pre-flight throws (an early abort must not strand it); `DragFrom`/`WheelAt` helpers on the existing `Smoke.Keys` P/Invokes; move by (-220,-160) asserted +/-2px; config content matches `^w=\d+ h=\d+ c=br$`; band click no-op; resize drags the RIGHT EDGE at mid-height - the window's top/bottom strips are region-clipped chrome where hit-testing misses (verification blocker; corners go to the manual list); wheel no-op; re-enter at persisted width. Execution gotcha: drag MOVEMENT must be injected via `mouse_event` MOVE|ABSOLUTE - `SetCursorPos` repositions the cursor without generating input events, so `WH_MOUSE_LL` never sees it and the armed drag never promotes (cost a 3-fail smoke run + instrumented-daemon probe to find).
8. **docs + versions**: SPEC §12 + stale-count fixes (intro/§9/§11 say 21 checks), ARCHITECTURE counts + hook bullet, README Controls, 2.1.0 in Cargo.toml + pip.lua descriptor; sweep `git grep -nE "2\.0\.0|21 (checks|end-to-end)|24 unit"`.
9. **verify + PR**: cargo test 41, clippy clean, release build (refresh ~165KB claims with the measured size; flag > 180KB), install + smoke 31/31, manual pass (corner resizes from each corner, wheel-volume + Ctrl+wheel still reach VLC, drag smoothness, 256px clamp feel), push + draft PR - no merge or tag without owner.

TDD throughout: tests land before implementations in tasks 1-3; every task ends with green tests and a signed commit.

## Post-review round (owner manual test, 2026-07-03)

Owner verified corners + wheel; three findings. Fixed: minimal look now stays live through a resize (region re-set per tick from start chrome offsets; convergence verifies the region BOX, not just presence) and close-in-PiP reopen overflow (stale state kept as a pending-restore record; the new player window is healed to the pre-PiP rect, state deleted only once the rect sticks - VLC's own Qt startup positioning loses the race). Smoke 31 -> 34. Deferred: sizing cursors over the band (needs an input-owning overlay window; VLC resets the cursor on every WM_SETCURSOR, so a hook can't win).
